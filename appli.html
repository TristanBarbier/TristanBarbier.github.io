Projet de cartographie

Le projet

Pendant les "vacances" (merci le Covid-19), notre projet était de faire une application interactive qui utilisait des données CSV (Comma Separated Values) pour produire une carte interactive, avec des graphiques représentant les données de nos tables CSV.

Explication

Il fallait tout d'abord choisir 3 questions auxquelles je voulais répondre:

  • Pourquoi les états qui ne font pas parti du territoire sont les plus riches?
  • 10 ans est-ce assez pour voir une évolution sur la richesse d'un pays/territoire?
  • Quelles projections peut-on faire pour les états pour les prochains 10 ans?

    • J'ai utilisé le revenu annuel, en moyenne de chaque État américain, et je les ai représentés sur un graphique en utilisant la librairie Python "folium". Ce que cette librairie fait, elle peut créer des cartes interactives, plus ou moins compliqué. Elle prend des tables (CSV par exemple) en entrée puis produit des cartes à partir de ces valeurs.Donc, il me fallait 2 tables: le revenu annuel de chaque État depuis au moins 2007, et les coordonnées GPS de chaque État afin de mettre les graphiques sur la carte correctement. Le problème le plus important était que je ne croyais pas avoir les clés nécessaires sur ma table pour pouvoir faire des graphiques de manière efficace. Mais deux fonctions m'ont permis de prendre le revenu qui était situé dans les clés des années, un par un, afin de produire des graphiques (fait avec une boucle). Cela est fait en utilisant des dictionnaires contenant des listes (donc la table que j'avais), et faire une boucle pour créer un JSON qui me permettrait de faire des graphiques lisibles.

    Les notions importantes

    L'OpenData

    Le terme Open Data désigne des données auxquelles n’importe qui peut accéder, que tout le monde peut utiliser ou partager. Dnas le cas du revenu mensuel, j'ai pu trouver la table sur data.gov et les coordonnes GPS sur public opendatasoft.

    Le standard JSON

    JSON est l'acronyme de Javascript Objet notation. Il s'agit d'un format de fichier open-standard permettant de stocker les données de manière organisée et lisible partout le monde, tout en y facilitant l'accès. Les données sont respresentés en utilisant des "clés" dans les tables, par exemple "years" contiendrait 2007, 2008, 2009, etc...

    J'ai choisi d'expliquer le programme "table Ops.py" puisque c'est la première chose que j'ai faite lors de ce projet. Comme tout programme python, j'ai importé les librairies dont j'avais besoin, pour pouvoir réaliser les opérations facilement. J'ai importé mes deux tables, celles de base. Ensuite, je devais réduire ma table pour contenir les données de 2007 à 2017, au lieu de 1984 à 2017. Je devais ensuite avoir une valeur "headers" qui me permettrait de réduire la table. Avec une boucle, j'ai donné les valeurs 2007 à 2017 à "headers". Ensuite, j'ai rajouté le string "State" pour garder les noms de chaque État. En utilisant la fonction "filter Column", j'ai réduit la table aux valeurs correspondantes, les revenus de 2007 à 2017, a le nom de chaque État. Il me manquait juste à combiner les tables, en utilisant une cle qu'ils avaient en commun: les états.
